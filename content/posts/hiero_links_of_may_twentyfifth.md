## Hiero Links For The Week Of May 25th

This week featured a lot of new and exciting updates within the Hiero Organization. 
Below are links to some of the things that happened this past week. **Check it out!**

---
**Anshul** updated the [ClusterFuzzlite](https://github.com/hiero-ledger/hiero-sdk-python/pull/2304) workflow in the Python SDK to only trigger when source code and .clusterfuzzlite/ are modified.

**Angelina Ceppaluni** added a blog post to the Hiero Website outlining the success and winners of the [Hello Future 2026 Apex Hackathon](https://hiero.org/blog/apex-2026-hiero-bounty-winners). 

**Manish Dait** updated [workflows](https://github.com/hiero-ledger/hiero-enterprise-java/pull/175) in the Hiero Enterprise Java repository to use pinned SHA’s to improve security and OpenSSF Scorecard compliance. 

**Fantomasa** updated [FreezeTransaction deserialization](https://github.com/hiero-ledger/hiero-sdk-js/pull/4095) in the JavaScript SDK to stop passing the deprecated endTime field through the public constructor path, preventing unnecessary deprecation warnings when reconstructing transactions from protobuffs.
